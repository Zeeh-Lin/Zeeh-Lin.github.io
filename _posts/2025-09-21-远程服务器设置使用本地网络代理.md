## 背景

也是鸟枪换炮，用上服务器了。

使用 Win 11 系统的PC远程连接 Ubuntu 20.04 系统的服务器。服务器上没有配置代理，无法连接 GitHub。需要配置服务器使用PC的代理。我本地使用 Clash Verge 进行代理。

找到一篇[博客](https://blog.csdn.net/weixin_44576482/article/details/128667675)解决这个问题，记录一下过程。

## 解决方案

参考这篇[博客](https://blog.csdn.net/weixin_44576482/article/details/128667675)

具体步骤如下：

1. 打开**本地**代理工具的局域网连接功能。并获得WLAN接口的IP地址。

2. 为 Windows 防火墙设置入站规则，开放本地的代理端口，我使用的端口是7897。

3. 对于git，在**服务器**配置

   ```bash
   # 重置仓库的http和https代理
   git config --unset http.proxy
   git config --unset https.proxy
   
   git config http.proxy http://192.168.21.54:7897
   git config https.proxy https://192.168.21.54:7897
   ```
